# CLAUDE.md

Project-specific conventions for this repo.

## Brand logo

The canonical InputMatter logo mark (the rounded orange-gradient square icon, no wordmark) lives at `src/assets/logo.svg`. It is the single source of truth for the logo everywhere in the app — don't recreate it with divs/gradients or duplicate it as a second file.

Use it via the shared `Logo` component at `src/app/components/Logo.tsx`:

```tsx
import Logo from "../components/Logo"; // adjust relative path to your file's location
<Logo size={30} />
```

Being an SVG, it scales to any size with no quality loss — just change the `size` prop, no separate asset needed per usage.

## Adding images / static assets

- Put reusable image assets in `src/assets/`. Don't use `src/imports/` for new, manually-added images — that directory is reserved for Figma-generated imports and is regenerated by Figma tooling.
- Import the image directly and use it like any other module, no special import scheme needed:
  ```tsx
  import shopAltImage from "../../assets/shop-alt.png";
  ```
- Downsize source images before committing them. Raw exports (AI-generated art, screenshots, design tool exports) are often 1MB+; an asset that only ever renders as a small icon or card image doesn't need to ship at full resolution. On macOS:
  ```bash
  sips -Z <max-dimension-px> file.png --out file.png
  ```
  Pick `max-dimension-px` at roughly 2x the largest size the image will actually render at (retina-safe without being wasteful).
- For an image sitting inside a fixed-size box (card, avatar, thumbnail), wrap it in a `relative overflow-hidden rounded-*` container and size the `<img>` with `w-full h-full object-cover`. This fills the box edge-to-edge consistently regardless of the box's aspect ratio, rather than `object-contain`, which can letterbox with visible gaps when the image and box ratios don't match.
- For an entity that may or may not have a real image (e.g. a shop's `logo_url`), fall back to a bundled placeholder asset with nullish coalescing, and keep the same sizing/fit classes for both the real and fallback image so they render consistently:
  ```tsx
  <img src={shop.logo_url ?? shopAltImage} alt={shop.name} className="w-full h-full object-cover" />
  ```
  See `ShopCard` in `src/app/pages/Welcome.tsx` for a working example of this pattern.
